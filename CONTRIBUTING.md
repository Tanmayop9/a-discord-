# Contributing to Discord Voice Mod

Thank you for your interest in contributing to Discord Voice Mod! This document provides guidelines and instructions for contributing.

## How to Contribute

### Reporting Bugs
- Check if the bug has already been reported in [Issues](https://github.com/Tanmayop9/a-discord-/issues)
- If not, create a new issue with:
  - Clear title and description
  - Steps to reproduce
  - Expected vs actual behavior
  - Device and Android version
  - Screenshots if applicable

### Suggesting Features
- Check existing issues and pull requests first
- Open a new issue with:
  - Clear description of the feature
  - Use case and benefits
  - Possible implementation approach

### Code Contributions

#### Setting Up Development Environment
1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/a-discord-.git
   ```
3. Create a feature branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```

#### Coding Standards
- Follow Java coding conventions
- Use meaningful variable and method names
- Add comments for complex logic
- Keep methods focused and concise
- Maintain consistent indentation (4 spaces)

#### Making Changes
1. Make your changes in the feature branch
2. Test thoroughly on different Android versions if possible
3. Ensure the app builds without errors
4. Update documentation if needed

#### Submitting Pull Requests
1. Commit your changes:
   ```bash
   git commit -m "Add: brief description of changes"
   ```
2. Push to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
3. Create a Pull Request with:
   - Clear title and description
   - Reference to related issues
   - Screenshots/videos for UI changes
   - List of changes made

## Code Review Process
- Maintainers will review your PR
- Address any requested changes
- Once approved, your PR will be merged

## Areas for Contribution
- **Voice Effects**: Implement new audio effects
- **UI/UX**: Improve user interface and experience
- **Performance**: Optimize audio processing
- **Documentation**: Improve README and code comments
- **Testing**: Add unit and integration tests
- **Bug Fixes**: Fix reported issues

## Questions?
Feel free to open an issue for questions or clarifications.

Thank you for contributing!
